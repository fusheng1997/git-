git与github工作简介：
	本地仓库由三部分组成，首先是工作目录，即本地电脑文件夹；其次是暂存区，临时保存改动，最后是HEAD，指向最后一次提交的结果

提交文件到GitHub
第一步：
	提交到暂存区：git add [filename]或者git add *（此处写要提交文件的后缀）
第二步:
	实际提交使用命令：git commit -m "备注信息"  此时只是提交到HEAD，还没有到远程仓库
第三步：
	git push origin master提交到远程仓库，master可以修改成任何你想要推送的分支

标签操作
-------------------------------------------------------
命令Git tag 查看标签；
命令Git tag <tagname> 可以创建一个本地标签；
命令Git push origin <tagname>可以推送一个本地标签；
命令git push origin --tags可以推送全部未推送过的本地标签；
命令git tag -d <tagname>可以删除一个本地标签；
命令git push origin :refs/tags/<tagname>可以删除一个远程标签。
-------------------------------------------------------





本地/远程仓库操作
-------------------------------------------------------
创建目录
指令：mkdir [目录名称]

初始化仓库
指令：git init


更新本地仓库至最新改动
指令 git pull [仓库别名] [分支或者标签]


添加远程仓库并与远程库建立链接(GitHub/gitee)
指令：git remote add [给远程库名取得名字][ssh或者http]
示例：git remote add github git@github.com:fusheng1997/user_code.git
示例：git remote add gitee git@gitee.com:fusheng1997/user_code.git

删除远程仓库名称
指令：git remote rm origin2
------------------------------------------------------





提取远程库到本地
------------------------------------------------------
查看远程库
指令：git remote
指令：git remote -v

提取远程仓库
指令：git fetch [远程库名] [分支]
指令：git merge 远程库名/分支

提取远程指定标签
指令：git clone --branch [标签] [远程仓库地址]
------------------------------------------------------




本地库上传远程库（一定要先添加远程仓库并建立连接）
------------------------------------------------------
添加文件到暂存区
指令：git add 文件名

提交到HEAD并备注信息
git commit -m "备注信息内容"

推送到远程库
指令：git push [远程库名] [要推送的分支名]
------------------------------------------------------







分支
------------------------------------------------------
显示远端相关分支
指令：git show-ref

修改远端分支
git push --set-upstream personal_origin D**(当前分支名)
